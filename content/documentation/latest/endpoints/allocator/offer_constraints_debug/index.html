USAGE

   /allocator/offer_constraints_debug

TL;DR;

Evaluates current framework offer constraints and returns results.

DESCRIPTION

This endpoint evaluates for each role of each framework a list of agents excluded from allocation by offer constraints.

Example: {"frameworks": { "0f4c63a9-be1e-4a90-9e11-d7bf0aa6c8ad-0017": { "excluded_by_attribute_constraints": { "role1": [ "0b1e7d60-dfbc-44c9-8222-48b57eca8637-S123", "654af69c-80f7-45ad-bcb3-c7c917f1811b-S045"], "role2": [] }}, "b0377da6-090d-4338-9e2e-bf6cf0f309b7-0011": {} }} In this example, two agents are excluded from allocation to the first framework (-0017) under the role “role1”, no agents are excluded from allocation to this framework under “role2”, and the second framework (-0011) has no offer constraints set.

AUTHENTICATION

This endpoint requires authentication iff HTTP authentication is enabled.

AUTHORIZATION

This endpoint skips frameworks for which the user is not authorizedto perform a VIEW_FRAMEWORK action.

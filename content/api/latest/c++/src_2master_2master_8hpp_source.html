1221  // We always want to rescind offers after the capacity heuristic. The
1222  // reason for this is the race between the allocator and the master:
1223  // it can happen that there are not enough free resources at the
1224  // allocator's disposal when it is notified about the quota request,
1225  // but at this point it's too late to rescind.

1227  // While rescinding, we adhere to the following rules:
1228  // * Rescind at least as many resources as there are in the quota request.
1229  // * Rescind all offers from an agent in order to make the potential
1230  // offer bigger, which increases the chances that a quota'ed framework
1231  // will be able to use the offer.

1233  // (but not guaranteed, due to fair sharing) that each framework in
1234  // the role for which quota is set gets an offer (`numF` is the
1235  // number of frameworks in the quota'ed role). Though this is not
1236  // strictly necessary, we think this will increase the debugability
1237  // and will improve user experience.

35 // The slave leverages the file system for a number of purposes:
36 //
37 // (1) For executor sandboxes and disk volumes.
38 //
39 // (2) For checkpointing metadata in order to support "slave
40 // recovery". That is, allow the slave to restart without
41 // affecting the tasks / executors.
42 //
43 // (3) To detect reboots (by checkpointing the boot id), in which
44 // case everything has died so no recovery should be attempted.
45 //
46 // (4) For checkpointing resources that persist across slaves.
47 // This includes things like persistent volumes and dynamic
48 // reservations.
49 //
50 // (5) For provisioning root filesystems for containers.
51 //
52 // (6) For CSI plugins to preserve data that persist across slaves.
53 //
54 // The file system layout is as follows:
55 //
56 // root ('--work_dir' flag)
57 // |-- containers
58 // | |-- <container_id> (sandbox)
59 // |-- slaves
60 // | |-- latest (symlink)
61 // | |-- <slave_id>
62 // | |-- frameworks
63 // | |-- <framework_id>
64 // | |-- executors
65 // | |-- <executor_id>
66 // | |-- runs
67 // | |-- latest (symlink)
68 // | |-- <container_id> (sandbox)
69 // |-- meta
70 // | |-- boot_id
71 // | |-- resources
72 // | | |-- resources.info
73 // | | |-- resources.target
74 // | | |-- resources_and_operations.state
75 // | |-- slaves
76 // | | |-- latest (symlink)
77 // | | |-- <slave_id>
78 // | | |-- slave.info
79 // | | |-- operations
80 // | | | |-- <operation_uuid>
81 // | | | |-- operation.updates
82 // | | |-- resource_providers
83 // | | | |-- <type>
84 // | | | |-- <name>
85 // | | | |-- latest (symlink)
86 // | | | |-- <resource_provider_id>
87 // | | | |-- resource_provider.state
88 // | | | |-- operations
89 // | | | |-- <operation_uuid>
90 // | | | |-- operation.updates
91 // | | |-- frameworks
92 // | | |-- <framework_id>
93 // | | |-- framework.info
94 // | | |-- framework.pid
95 // | | |-- executors
96 // | | |-- <executor_id>
97 // | | |-- executor.info
98 // | | |-- runs
99 // | | |-- latest (symlink)
100 // | | |-- <container_id> (sandbox)
101 // | | |-- executor.sentinel (if completed)
102 // | | |-- pids
103 // | | | |-- forked.pid
104 // | | | |-- libprocess.pid
105 // | | |-- tasks
106 // | | |-- <task_id>
107 // | | |-- task.info
108 // | | |-- task.updates
109 // | |-- volume_gid_manager
110 // | |-- volume_gids
111 // |-- volumes
112 // | |-- roles
113 // | |-- <role>
114 // | |-- <persistence_id> (persistent volume)
115 // |-- provisioner
116 // |-- csi
